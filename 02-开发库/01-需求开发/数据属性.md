## 2. 数据属性定义

### 2.1 信件

**实体说明**：信件主信息，记录群众来信的完整内容。

| 属性名称 | 属性类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 信件编号 | 编号 | 是 | 系统自动生成的唯一标识 |
| 来信人姓名 | 姓名 | 是 | 来信人真实姓名 |

| 来信人联系地址 | 地址 | 否 | 来信人提供的联系地址 |
| 来信人归属地址 | 地址 | 否 | 根据身份证解析的归属地 |
| 留言标题 | 标题 | 是 | 信件标题 |
| 留言内容 | 正文 | 是 | 信件详细内容 |
| 留言备注 | 备注 | 否 | 内部备注信息 |
| 留言时间 | 时间 | 是 | 来信提交时间 |
| 信件状态 | 状态 | 是 | 当前处理状态 |
| 留言来源 | 来源 | 是 | 厅长信箱/局长信箱/信访件/12389等 |
| 留言类型 | 类型 | 否 | 咨询/投诉/建议/举报等 |
| 业务类型 | 类型 | 否 | 具体业务分类 |
| 案件指向 | 类型 | 否 | 是否指向具体案件 |
| 是否重复件 | 标记 | 否 | 是否为重复来信 |
| 是否加急 | 标记 | 否 | 是否为加急信件 |

## 3. 数据关系

### 3.1 核心关系图

```
信件（主实体）
    │
    ├── 办理单位（一对多）
    │       ├── 信件回复（一对一）
    │       ├── 信件办理（一对一）
    │       └── 回复处理流程（一对多）
    │
    ├── 标签关联（一对多） → 标签（多对一）
    │
    ├── 已读记录（一对多）
    │
    ├── 收藏记录（一对多）
    │
    ├── 预警关联（一对多） → 预警牌配置（多对一）
    │
    ├── 回访信息（一对一）
    │
    └── 特色信息（一对一）
```

### 3.2 关系说明

| 主实体 | 关联实体 | 关系类型 | 说明 |
|--------|----------|----------|------|
| 信件 | 办理单位 | 一对多 | 一封信件可指派多个办理单位（主办+督办+协办） |
| 办理单位 | 信件回复 | 一对一 | 每个办理单位对应一个回复 |
| 办理单位 | 信件办理 | 一对一 | 每个办理单位对应一个办理记录 |
| 办理单位 | 回复处理流程 | 一对多 | 每个办理单位可有多次审核流程记录 |
| 信件 | 标签关联 | 一对多 | 一封信件可有多个标签 |
| 标签关联 | 标签 | 多对一 | 多个信件可关联同一标签 |
| 信件 | 预警关联 | 一对多 | 一封信件可有多个单位的预警记录 |
| 预警关联 | 预警牌配置 | 多对一 | 多个信件使用同一预警配置 |
