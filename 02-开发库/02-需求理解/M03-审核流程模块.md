## 1. 模块概述

### 1.1 模块定位

审核流程模块是gaxx-pro信件处理系统的核心业务流程控制模块，负责信件办理结果的审核与流转控制。该模块确保信件回复内容的质量，规范信件处理流程，保障群众诉求得到有效解决。

### 1.2 业务价值

- **质量保障**：通过多级审核机制，确保回复内容的准确性、完整性和规范性
- **流程规范**：明确审核流程节点和操作权限，规范处理行为
- **效率提升**：提供灵活的审核路径（一审+终审、直接终审），适应不同业务场景
- **责任追溯**：完整的流程记录，支持操作追溯和责任认定

### 1.3 模块范围

本模块包含11个功能点，覆盖审核流程的各个环节：

| 功能编号 | 功能名称 | 功能类型 | 优先级 |

## 2. 功能点详细分析

### 2.1 F03-01 回复信件

#### 功能描述

主办单位完成信件调查处理后，提交回复内容进入审核流程。回复内容需经过审核通过后方可发送给网民。

#### 操作角色

| 角色 | 操作权限 | 说明 |
|------|----------|------|
| 主办单位 | 可提交回复 | 负责信件办理并提交回复内容 |

#### 输入

| 参数名称 | 参数类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 信件编号 | 编号 | 是 | 信件唯一标识 |
| 回复内容 | 正文 | 是 | 给网民的回复正文 |
| 上级回复内容 | 正文 | 否 | 上级部门的回复内容（如有） |
| 是否结案 | 标记 | 否 | 0否/1是，是否建议结案 |
| 回复附件 | 文件 | 否 | 回复相关附件材料 |

#### 输出

| 输出项 | 说明 |
|--------|------|
| 回复记录 | 创建的回复记录信息 |
| 流程状态 | 信件状态变更为"待审核" |
| 流程日志 | 记录提交操作日志 |

#### 业务规则

1. **前置状态检查**：信件状态必须为"受理中"
2. **权限验证**：必须是信件的主办单位才能提交回复
3. **内容校验**：回复内容不能为空，需符合格式规范
4. **唯一性校验**：同一办理单位同一信件只能有一个有效回复记录
5. **状态变更**：提交成功后信件状态变更为"待审核"
6. **时限检查**：提交时检查是否超期，超期需说明原因

#### 前置条件

- 信件已指派主办单位
- 当前用户为主办单位人员
- 信件状态为"受理中"
- 已完成调查核实工作

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 信件状态异常 | 返回错误提示"信件当前状态不允许提交回复" |
| 权限不足 | 返回错误提示"无权限操作此信件" |
| 内容为空 | 返回错误提示"回复内容不能为空" |
| 重复提交 | 返回错误提示"已存在待审核回复，请勿重复提交" |
| 系统异常 | 记录错误日志，返回"系统异常，请稍后重试" |

#### 操作角色

| 角色 | 操作权限 | 说明 |
|------|----------|------|
| 主办单位 | 可提交办理表 | 负责填写办理详情 |

#### 输入

| 参数名称 | 参数类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 办理单位编号 | 编号 | 是 | 办理单位记录编号 |
| **回访信息** | | | |
| 是否回访 | 标记 | 是 | 是否已回访群众 |
| 未回访原因 | 原因 | 否 | 未回访的具体原因 |
| 回访情况 | 说明 | 否 | 回访情况详细说明 |
| 群众电话回访反映 | 说明 | 否 | 群众电话回访反映内容 |
| 是否需要再次回复 | 标记 | 否 | 是否需要再次回复 |
| **满意度评价** | | | |
| 诉求解决满意度 | 评分 | 否 | 满意/基本满意/不满意 |
| 响应速度满意度 | 评分 | 否 | 满意/基本满意/不满意 |
| 服务态度满意度 | 评分 | 否 | 满意/基本满意/不满意 |
| 办事能力满意度 | 评分 | 否 | 满意/基本满意/不满意 |
| 跟进服务满意度 | 评分 | 否 | 满意/基本满意/不满意 |
| 整体满意度 | 评分 | 否 | 满意/基本满意/不满意 |
| **调查情况** | | | |
| 说法是否一致 | 标记 | 否 | 公安机关和群众说法是否一致 |
| 调查情况 | 状态 | 否 | 调查完成情况 |
| 无法调查原因 | 原因 | 否 | 无法开展调查的原因 |
| 解决情况 | 状态 | 否 | 问题解决情况说明 |
| 查实情况 | 状态 | 否 | 情况核实结果 |
| **接访信息** | | | |
| 是否接访 | 标记 | 否 | 是否进行接访 |
| 接访方式 | 方式 | 否 | 接访方式（电话/当面/书面等） |
| 接访领导 | 人员 | 否 | 接访领导姓名 |
| 接访时间 | 时间 | 否 | 接访时间 |
| **办理人信息** | | | |
| 办理人姓名 | 人员 | 是 | 主办民警姓名 |
| 办理人身份证 | 身份证号 | 是 | 主办民警身份证号 |
| 办理人单位 | 单位 | 是 | 主办民警所在单位 |

#### 输出

| 输出项 | 说明 |
|--------|------|
| 办理记录 | 创建的办理记录信息 |
| 关联状态 | 办理单位记录关联办理表 |

#### 业务规则

1. **关联校验**：办理单位编号必须有效且属于当前用户所在单位
2. **必填校验**：办理人信息为必填项
3. **逻辑校验**：如选择"未回访"，必须填写未回访原因
4. **满意度完整性**：如已回访，建议填写满意度评价
5. **接访校验**：如选择"已接访"，必须填写接访方式和接访领导
6. **审核信息**：重大案件需呈法制或督察部门审核

#### 前置条件

- 信件已指派主办单位
- 当前用户为主办单位人员
- 办理单位状态为"已指派"

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 办理单位不存在 | 返回错误提示"办理单位记录不存在" |
| 权限不足 | 返回错误提示"无权限操作此办理单位" |
| 必填项缺失 | 返回错误提示具体缺失字段 |
| 数据格式错误 | 返回错误提示具体格式错误项 |

### 2.3 F03-03 下级单位退回-无法处理

#### 功能描述

办理单位因客观原因无法处理该信件时，可申请退回上级单位。退回需说明具体原因，上级单位决定后续处理方式。

#### 操作角色

| 角色 | 操作权限 | 说明 |
|------|----------|------|
| 主办单位 | 可申请退回 | 因无法处理申请退回 |

#### 输入

| 参数名称 | 参数类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 信件编号 | 编号 | 是 | 信件唯一标识 |
| 退回原因 | 原因 | 是 | 无法处理的具体原因 |
| 退回类型 | 类型 | 是 | 退回类型（职责范围外/资源不足/信息不全等） |
| 备注说明 | 说明 | 否 | 补充说明材料 |

#### 输出

| 输出项 | 说明 |
|--------|------|
| 流程状态 | 信件退回至上级单位 |
| 流程记录 | 记录退回操作日志 |
| 通知消息 | 通知上级单位处理 |

#### 业务规则

1. **状态检查**：信件状态必须为"受理中"
2. **权限校验**：只有主办单位可申请退回
3. **原因必填**：退回原因必须详细填写
4. **上级接收**：退回后信件流转至指派来源单位
5. **状态变更**：信件状态变更为"待处理"（回退到上级）
6. **通知机制**：退回后自动通知上级单位

#### 前置条件

- 信件已指派办理单位
- 当前用户为主办单位人员
- 信件状态为"受理中"
- 尚未提交回复

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 状态不允许退回 | 返回错误提示"已提交回复的信件不能退回" |
| 权限不足 | 返回错误提示"只有主办单位可申请退回" |
| 原因未填写 | 返回错误提示"退回原因不能为空" |
| 上级单位不存在 | 返回错误提示"无法找到上级接收单位" |

### 2.4 F03-04 上级单位退回-重新修改

#### 功能描述

审核单位审核回复内容后，认为需要修改完善，将信件退回主办单位要求修改回复内容。主办单位修改后可再次提交审核。

#### 操作角色

| 角色 | 操作权限 | 说明 |
|------|----------|------|
| 接收单位 | 可退回修改 | 终审时退回修改 |
| 督办单位 | 可退回修改 | 一审时退回修改 |

#### 输入

| 参数名称 | 参数类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 信件编号 | 编号 | 是 | 信件唯一标识 |
| 退回意见 | 意见 | 是 | 要求修改的具体意见 |
| 修改要点 | 说明 | 否 | 需要重点修改的内容 |

#### 输出

| 输出项 | 说明 |
|--------|------|
| 流程状态 | 信件状态变更为"重新修改" |
| 流程记录 | 记录退回操作日志 |
| 通知消息 | 通知主办单位修改 |

#### 业务规则

1. **状态检查**：信件状态必须为"待审核"
2. **权限校验**：接收单位或督办单位可执行
3. **意见必填**：退回意见必须详细说明需要修改的内容
4. **状态变更**：信件状态变更为"重新修改"
5. **回复状态**：回复记录状态变更为"已退回"
6. **通知机制**：退回后自动通知主办单位

#### 前置条件

- 信件状态为"待审核"
- 当前用户为接收单位或督办单位人员
- 存在待审核的回复内容

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 状态不允许 | 返回错误提示"当前状态不允许退回修改" |
| 权限不足 | 返回错误提示"无审核权限" |
| 意见未填写 | 返回错误提示"退回意见不能为空" |
| 无回复内容 | 返回错误提示"未找到待审核回复内容" |

审核单位认为回复内容存在根本性问题，需要重新调查处理后再次提交。此操作比"重新修改"更严格，要求主办单位重新开展调查工作。

#### 操作角色

| 角色 | 操作权限 | 说明 |
|------|----------|------|
| 接收单位 | 可退回重办 | 终审时退回重新办理 |
| 督办单位 | 可退回重办 | 一审时退回重新办理 |

#### 输入

| 参数名称 | 参数类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 信件编号 | 编号 | 是 | 信件唯一标识 |
| 退回意见 | 意见 | 是 | 要求重新办理的具体意见 |
| 重办原因 | 原因 | 是 | 重新办理的原因分类 |
| 备注说明 | 说明 | 否 | 补充说明 |

#### 输出

| 输出项 | 说明 |
|--------|------|
| 流程状态 | 信件状态变更为"重新办理" |
| 流程记录 | 记录退回操作日志 |
| 通知消息 | 通知主办单位重新办理 |

#### 业务规则

1. **状态检查**：信件状态必须为"待审核"
2. **权限校验**：接收单位或督办单位可执行
3. **意见必填**：退回意见必须详细说明问题所在
4. **原因分类**：需选择重新办理的原因类型
5. **状态变更**：信件状态变更为"重新办理"
6. **办理表状态**：原办理表记录标记为无效
7. **通知机制**：退回后自动通知主办单位

#### 前置条件

- 信件状态为"待审核"
- 当前用户为接收单位或督办单位人员
- 存在待审核的回复内容

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 状态不允许 | 返回错误提示"当前状态不允许退回重办" |
| 权限不足 | 返回错误提示"无审核权限" |
| 意见未填写 | 返回错误提示"退回意见不能为空" |
| 原因未选择 | 返回错误提示"请选择重新办理原因" |

### 2.6 F03-06 审核一审通过

#### 功能描述

督办单位对主办单位提交的回复内容进行一级审核，审核通过后进入终审阶段。

#### 操作角色

| 角色 | 操作权限 | 说明 |
|------|----------|------|
| 督办单位 | 可一审通过 | 负责一级审核 |
| 接收单位 | 可一审通过（代理） | 无督办单位时可代理一审 |

#### 输入

| 参数名称 | 参数类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 信件编号 | 编号 | 是 | 信件唯一标识 |
| 审核意见 | 意见 | 否 | 一审审核意见 |
| 审核评分 | 评分 | 否 | 回复质量评分 |

#### 输出

| 输出项 | 说明 |
|--------|------|
| 流程状态 | 进入终审阶段，状态保持"待审核" |
| 流程记录 | 记录一审通过操作 |
| 审核标记 | 回复记录标记一审通过 |
| 通知消息 | 通知接收单位进行终审 |

#### 业务规则

1. **状态检查**：信件状态必须为"待审核"
2. **权限校验**：督办单位或接收单位可执行
3. **审核标记**：回复记录标记"一审已通过"
4. **流程推进**：信件进入终审阶段
5. **通知机制**：自动通知接收单位进行终审
6. **无督办单位场景**：如无督办单位，接收单位可直接进行终审

#### 前置条件

- 信件状态为"待审核"
- 当前用户为督办单位或接收单位人员
- 存在待审核的回复内容
- 回复内容尚未一审通过

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 状态不允许 | 返回错误提示"当前状态不允许一审" |
| 权限不足 | 返回错误提示"无一审审核权限" |
| 已一审通过 | 返回错误提示"回复已一审通过，请进行终审" |
| 无回复内容 | 返回错误提示"未找到待审核回复内容" |

### 2.7 F03-07 审核终审通过

#### 功能描述

接收单位对回复内容进行最终审核，审核通过后信件进入"待回复"状态，准备正式回复网民。

#### 操作角色

| 角色 | 操作权限 | 说明 |
|------|----------|------|
| 接收单位 | 可终审通过 | 负责最终审核 |

#### 输入

| 参数名称 | 参数类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 信件编号 | 编号 | 是 | 信件唯一标识 |
| 终审意见 | 意见 | 否 | 终审审核意见 |
| 是否需要修改 | 标记 | 否 | 是否需要小幅修改后回复 |

#### 输出

| 输出项 | 说明 |
|--------|------|
| 流程状态 | 信件状态变更为"待回复" |
| 流程记录 | 记录终审通过操作 |
| 审核标记 | 回复记录标记终审通过 |
| 通知消息 | 通知主办单位准备回复 |

#### 业务规则

1. **状态检查**：信件状态必须为"待审核"，且一审已通过（如有督办单位）
2. **权限校验**：只有接收单位可执行终审
3. **一审前置**：如有督办单位，需先完成一审通过
4. **审核标记**：回复记录标记"终审已通过"
5. **状态变更**：信件状态变更为"待回复"
6. **时间记录**：记录终审通过时间

#### 前置条件

- 信件状态为"待审核"
- 当前用户为接收单位人员
- 存在待审核的回复内容
- 如有督办单位，需先完成一审通过

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 状态不允许 | 返回错误提示"当前状态不允许终审" |
| 权限不足 | 返回错误提示"只有接收单位可终审" |
| 未完成一审 | 返回错误提示"请先完成一审审核" |
| 无回复内容 | 返回错误提示"未找到待审核回复内容" |

### 2.8 F03-08 直接终审通过

#### 功能描述

在特定条件下，接收单位可跳过一级审核，直接进行终审通过。适用于紧急信件或无督办单位的场景。

#### 操作角色

| 角色 | 操作权限 | 说明 |
|------|----------|------|
| 接收单位 | 可直接终审 | 负责直接终审 |

#### 输入

| 参数名称 | 参数类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 信件编号 | 编号 | 是 | 信件唯一标识 |
| 终审意见 | 意见 | 否 | 终审审核意见 |
| 直接终审原因 | 原因 | 否 | 直接终审的原因说明 |

#### 输出

| 输出项 | 说明 |
|--------|------|
| 流程状态 | 信件状态变更为"待回复" |
| 流程记录 | 记录直接终审通过操作 |
| 审核标记 | 回复记录标记终审通过（跳过一审） |
| 通知消息 | 通知主办单位准备回复 |

#### 业务规则

1. **状态检查**：信件状态必须为"待审核"
2. **权限校验**：只有接收单位可执行直接终审
3. **适用场景**：
   - 无督办单位的信件
   - 紧急信件需快速回复
   - 回复内容明确无误
4. **审核标记**：回复记录标记"直接终审通过"
5. **状态变更**：信件状态变更为"待回复"
6. **流程记录**：明确记录跳过一审的原因

#### 前置条件

- 信件状态为"待审核"
- 当前用户为接收单位人员
- 存在待审核的回复内容
- 符合直接终审的条件

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 状态不允许 | 返回错误提示"当前状态不允许终审" |
| 权限不足 | 返回错误提示"只有接收单位可终审" |
| 有督办单位且非紧急 | 返回错误提示"请按正常流程先完成一审" |
| 无回复内容 | 返回错误提示"未找到待审核回复内容" |

### 2.9 F03-09 审核不通过

#### 功能描述

审核单位审核回复内容后，认为回复严重不合格，驳回回复内容。审核不通过后需要选择后续处理方式（重新修改或重新办理）。

#### 操作角色

| 角色 | 操作权限 | 说明 |
|------|----------|------|
| 接收单位 | 可审核驳回 | 终审时可驳回 |
| 督办单位 | 可审核驳回 | 一审时可驳回 |

#### 输入

| 参数名称 | 参数类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 信件编号 | 编号 | 是 | 信件唯一标识 |
| 驳回原因 | 原因 | 是 | 审核不通过的具体原因 |
| 后续处理 | 类型 | 是 | 后续处理方式（重新修改/重新办理） |
| 驳回说明 | 说明 | 否 | 详细驳回说明 |

#### 输出

| 输出项 | 说明 |
|--------|------|
| 流程状态 | 信件状态变更为"审核不通过" |
| 流程记录 | 记录审核驳回操作 |
| 通知消息 | 通知主办单位处理 |

#### 业务规则

1. **状态检查**：信件状态必须为"待审核"
2. **权限校验**：接收单位或督办单位可执行
3. **原因必填**：驳回原因必须详细说明
4. **后续处理**：需明确后续处理方式
5. **状态变更**：信件状态变更为"审核不通过"
6. **流程推进**：根据后续处理方式自动推进到相应状态

#### 前置条件

- 信件状态为"待审核"
- 当前用户为接收单位或督办单位人员
- 存在待审核的回复内容

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 状态不允许 | 返回错误提示"当前状态不允许审核" |
| 权限不足 | 返回错误提示"无审核权限" |
| 原因未填写 | 返回错误提示"驳回原因不能为空" |
| 后续处理未选择 | 返回错误提示"请选择后续处理方式" |

| 9 | 重新修改 | 要求修改回复内容 | 主办单位修改后提交 |
| 10 | 重新办理 | 要求重新调查处理 | 主办单位重新办理 |

## 6. 用户故事

### 6.1 主办单位用户故事

**US-M03-01：提交回复内容**

作为一名主办单位工作人员，我希望能够提交信件回复内容并进入审核流程，以便我的工作成果能够得到审核确认。

验收标准：
- 能够填写完整的回复内容
- 提交后信件状态变为"待审核"
- 系统记录提交时间和操作人
- 提交后收到确认通知

**US-M03-02：提交办理详情**

作为一名主办单位工作人员，我希望能够提交办理详情（包括走访、满意度等信息），以便审核人员了解办理过程。

验收标准：
- 能够填写完整的办理表信息
- 办理人信息为必填项
- 系统自动关联对应的信件和办理单位

**US-M03-03：处理退回修改**

作为一名主办单位工作人员，当我的回复被退回修改时，我希望能看到具体的修改意见，以便我能针对性地修改回复内容。

验收标准：
- 能看到退回意见和修改要点
- 修改后可再次提交审核
- 系统记录修改次数

**US-M03-04：申请无法处理**

作为一名主办单位工作人员，当我遇到无法处理的信件时，我希望能够申请退回上级，以便上级重新安排处理。

验收标准：
- 能够说明无法处理的原因
- 退回后收到确认通知
- 上级能收到退回提醒

### 6.2 督办单位用户故事

**US-M03-05：一级审核回复**

作为一名督办单位工作人员，我希望能够审核主办单位提交的回复内容，以便确保回复质量达标。

验收标准：
- 能查看回复内容和办理详情
- 能选择通过或不通过
- 不通过时能填写驳回意见
- 系统记录审核时间和结果

### 6.3 接收单位用户故事

**US-M03-06：最终审核回复**

作为一名接收单位工作人员，我希望能够进行终审审核，以便确认回复内容可以发送给网民。

验收标准：
- 能查看一审意见（如有）
- 能选择终审通过或不通过
- 通过后信件进入待回复状态
- 系统记录终审时间和结果

**US-M03-07：直接终审**

作为一名接收单位工作人员，在紧急情况下我希望能够跳过一审直接终审，以便快速回复网民。

验收标准：
- 能执行直接终审操作
- 系统记录跳过一审的原因
