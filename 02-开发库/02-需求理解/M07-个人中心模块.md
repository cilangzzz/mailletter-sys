## 1. 模块概述

### 1.1 模块定位

个人中心模块是gaxx-pro信件处理系统的辅助功能模块，为用户提供个性化的信件管理功能。该模块主要支持用户对信件的收藏和已读状态管理，帮助用户快速定位和追踪关注的信件，提升工作效率。

### 1.2 功能范围

| 功能编号 | 功能名称 | 优先级 | 说明 |
|----------|----------|--------|------|
| F07-01 | 收藏信件 | P2 | 用户收藏关注的信件 |
| F07-02 | 取消收藏信件 | P2 | 用户取消已收藏的信件 |
| F07-03 | 标记已读 | P2 | 用户标记信件为已读状态 |
| F07-04 | 查看我的收藏 | P2 | 查看个人收藏的信件列表 |

### 1.3 用户角色

| 角色 | 说明 | 可使用功能 |
|------|------|------------|
| 管理员 | 系统管理人员 | F07-01~04 |
| 接收单位 | 信件接收单位人员 | F07-01~04 |
| 办理单位 | 信件办理单位人员 | F07-01~04 |
| 督办单位 | 信件督办单位人员 | F07-01~04 |

所有角色均可使用本模块的全部功能。

### 1.4 业务场景

- **场景1：** 办理人员在处理信件过程中，发现某信件需要长期跟进，将其收藏以便快速查找
- **场景2：** 接收单位人员在审核大量信件后，需要快速识别哪些信件已经查看过
- **场景3：** 督办人员需要追踪多个重点信件，通过收藏功能建立个人关注列表
- **场景4：** 用户定期查看收藏列表，跟进关注的信件处理进度

## 2. 功能点详细分析

### 2.1 F07-01 收藏信件

#### 功能描述

用户将关注的信件添加到个人收藏列表，方便后续快速查找和跟进。

#### 操作角色

所有角色（管理员、接收单位、办理单位、督办单位）

#### 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 信件编号 | 编号 | 是 | 要收藏的信件唯一标识 |

#### 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 收藏记录 | 记录 | 新创建的收藏记录 |
| 操作结果 | 结果 | 收藏成功/失败提示 |

#### 业务规则

1. **唯一性规则**：同一用户对同一信件只能收藏一次，重复收藏时返回"已收藏"提示
2. **状态规则**：收藏状态为"正常"（状态值=1）
3. **权限规则**：用户只能收藏自己有权查看的信件
4. **记录规则**：系统自动记录收藏时间

#### 前置条件

1. 用户已登录系统
2. 用户具有查看该信件的权限
3. 信件存在且未被删除
4. 用户未收藏过该信件

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 信件不存在 | 返回错误提示"信件不存在或已删除" |
| 信件已删除 | 返回错误提示"信件已删除，无法收藏" |
| 无查看权限 | 返回错误提示"无权限收藏该信件" |
| 已收藏过 | 返回提示"该信件已在收藏列表中" |
| 系统异常 | 返回错误提示"收藏失败，请稍后重试" |

---

### 2.2 F07-02 取消收藏信件

#### 功能描述

用户将已收藏的信件从个人收藏列表中移除，取消关注状态。

#### 操作角色

所有角色（管理员、接收单位、办理单位、督办单位）

#### 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 信件编号 | 编号 | 是 | 要取消收藏的信件唯一标识 |

#### 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 操作结果 | 结果 | 取消成功/失败提示 |

#### 业务规则

1. **状态变更规则**：取消收藏不删除记录，而是将收藏状态改为"已取消"（状态值=2）
2. **权限规则**：只能取消自己收藏的信件
3. **记录保留规则**：取消后保留收藏记录，支持历史查询和统计分析

#### 前置条件

1. 用户已登录系统
2. 该信件已被用户收藏（状态为正常）
3. 收藏记录存在

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 信件未收藏 | 返回提示"该信件未在收藏列表中" |
| 收藏记录不存在 | 返回错误提示"收藏记录不存在" |
| 非本人收藏 | 返回错误提示"无权限操作他人收藏" |
| 系统异常 | 返回错误提示"取消收藏失败，请稍后重试" |

---

### 2.3 F07-03 标记已读

#### 功能描述

用户将信件标记为已读状态，用于区分已查看和未查看的信件，便于批量处理时识别。

#### 操作角色

所有角色（管理员、接收单位、办理单位、督办单位）

#### 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 信件编号 | 编号 | 是 | 要标记已读的信件唯一标识 |

#### 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 已读记录 | 记录 | 新创建或更新的已读记录 |
| 操作结果 | 结果 | 标记成功/失败提示 |

#### 业务规则

1. **首次标记规则**：首次标记时创建已读记录
2. **重复标记规则**：重复标记时更新已读状态，不报错
3. **状态值规则**：已读状态值为1（已读），未读状态值为0
4. **权限规则**：只能标记自己有权查看的信件
5. **自动标记规则**：用户查看信件详情时系统可自动标记已读（可选功能）

#### 前置条件

1. 用户已登录系统
2. 用户具有查看该信件的权限
3. 信件存在且未被删除

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 信件不存在 | 返回错误提示"信件不存在或已删除" |
| 信件已删除 | 返回错误提示"信件已删除，无法标记" |
| 无查看权限 | 返回错误提示"无权限标记该信件" |
| 系统异常 | 返回错误提示"标记失败，请稍后重试" |

---

### 2.4 F07-04 查看我的收藏

#### 功能描述

用户查看个人收藏的信件列表，可按条件筛选和排序，便于快速定位关注的信件。

#### 操作角色

所有角色（管理员、接收单位、办理单位、督办单位）

#### 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 筛选条件 | 条件 | 否 | 可选的筛选条件 |
| 分页参数 | 参数 | 否 | 页码、每页条数 |
| 排序参数 | 参数 | 否 | 排序字段、排序方式 |

**可选筛选条件**：
- 信件状态（待处理/受理中/待审核/待回复/已完成等）
- 留言类型
- 业务类型
- 收藏时间范围
- 关键词搜索（标题/内容）

#### 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 收藏信件列表 | 列表 | 分页后的收藏信件列表 |
| 总数量 | 数字 | 符合条件的收藏总数 |
| 分页信息 | 信息 | 当前页、总页数、每页条数 |

**列表项信息**：
- 信件编号
- 留言标题
- 信件状态
- 收藏时间
- 办理进度（可选）

#### 业务规则

1. **范围规则**：只显示当前用户收藏的信件（状态为正常）
2. **排序规则**：默认按收藏时间倒序排列（最近收藏在前）
3. **分页规则**：默认每页显示20条记录
4. **权限规则**：只显示用户有权查看的信件

#### 前置条件

1. 用户已登录系统
2. 用户有收藏记录

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 无收藏记录 | 返回空列表，提示"暂无收藏信件" |
| 系统异常 | 返回错误提示"查询失败，请稍后重试" |

## 3. 数据实体关联

### 3.1 相关数据实体

本模块主要涉及两个数据实体：

| 实体名称 | 说明 | 优先级 | 关联功能 |
|----------|------|--------|----------|
| 已读记录 | 用户已读信件的记录 | P2 | F07-03 |
| 收藏记录 | 用户收藏信件的记录 | P2 | F07-01, F07-02, F07-04 |

### 3.2 已读记录实体属性

| 属性名称 | 属性类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 用户 | 人员 | 是 | 读取信件的用户 |
| 关联信件 | 关联 | 是 | 对应的信件编号 |
| 已读状态 | 状态 | 是 | 0未读/1已读 |

### 3.3 收藏记录实体属性

| 属性名称 | 属性类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 用户 | 人员 | 是 | 收藏信件的用户 |
| 关联信件 | 关联 | 是 | 对应的信件编号 |
| 收藏时间 | 时间 | 是 | 收藏操作时间 |
| 收藏状态 | 状态 | 是 | 1正常/2已取消 |

### 3.4 数据关系图

```
用户（操作主体）
    │
    ├── 已读记录（一对多）
    │       │
    │       └── 信件（多对一）
    │
    └── 收藏记录（一对多）
            │
            └── 信件（多对一）
```

### 3.5 与核心实体的关系

| 主实体 | 关联实体 | 关系类型 | 说明 |
|--------|----------|----------|------|
| 信件 | 已读记录 | 一对多 | 一封信件可被多个用户标记已读 |
| 信件 | 收藏记录 | 一对多 | 一封信件可被多个用户收藏 |
| 用户 | 已读记录 | 一对多 | 一个用户可标记多封信件已读 |
| 用户 | 收藏记录 | 一对多 | 一个用户可收藏多封信件 |

## 4. 用户故事

### 4.1 收藏信件（F07-01）

**故事编号：** US07-01  
**作为：** 办理单位人员  
**我想要：** 收藏我需要长期跟进的信件  
**以便于：** 快速找到这些信件，随时查看处理进度

**验收条件：**
- 在信件详情页有收藏按钮
- 点击后信件进入我的收藏列表
- 收藏成功有提示反馈
- 已收藏的信件显示收藏状态标识

### 4.2 取消收藏信件（F07-02）

**故事编号：** US07-02  
**作为：** 督办单位人员  
**我想要：** 取消不再需要关注的信件收藏  
**以便于：** 保持收藏列表整洁，只保留当前关注的信件

**验收条件：**
- 在收藏列表可取消收藏
- 在信件详情页可取消收藏
- 取消成功有提示反馈
- 取消后信件从收藏列表消失

### 4.3 标记已读（F07-03）

**故事编号：** US07-03  
**作为：** 接收单位人员  
**我想要：** 标记我已查看过的信件  
**以便于：** 在大量信件中区分哪些已处理、哪些待处理

**验收条件：**
- 在信件列表有已读/未读状态标识
- 点击信件或手动标记后变为已读状态
- 未读信件有醒目标识（如红点、加粗）
- 可批量标记已读

### 4.4 查看我的收藏（F07-04）

**故事编号：** US07-04  
**作为：** 办理单位人员  
**我想要：** 查看我收藏的所有信件列表  
**以便于：** 快速定位并跟进我关注的信件

**验收条件：**
- 有专门的收藏列表入口
- 列表显示信件基本信息和收藏时间
- 支持筛选和排序
- 可点击进入信件详情
- 支持分页显示

## 6. 界面设计建议

### 6.1 收藏功能入口

- 信件详情页右上角：收藏图标（星形/书签）
- 信件列表每行：收藏快捷按钮
- 列表批量操作栏：批量收藏按钮

### 6.2 已读状态显示

- 信件列表未读信件：标题加粗 + 红点标识
- 信件列表已读信件：常规显示
- 状态切换：点击自动切换为已读

### 6.3 收藏列表页面

- 左侧菜单：我的收藏入口
- 列表区域：卡片或表格形式展示收藏信件
- 顶部筛选栏：状态筛选、时间筛选、关键词搜索
- 每行操作：取消收藏、查看详情

## 7. 技术实现要点

### 7.1 数据存储

- 已读记录：独立表存储，支持快速查询
- 收藏记录：独立表存储，状态字段支持取消后保留

### 7.2 性能优化

- 收藏列表：支持分页，避免全量加载
- 已读状态：信件列表查询时关联已读表，批量返回状态
- 缓存策略：用户收藏列表可考虑缓存

### 7.3 并发处理

- 收藏/取消收藏：需处理并发操作（用户快速点击）
- 已读标记：重复标记不报错
