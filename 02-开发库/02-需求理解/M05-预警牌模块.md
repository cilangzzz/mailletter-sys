## 1. 模块概述

### 1.1 模块定位

预警牌模块是gaxx-pro信件处理系统的时限管理核心模块，负责信件办理时限的配置与监控。通过预设时限规则，系统自动计算信件的办理进度，对即将超期或已超期的信件进行预警提醒，确保信件按时办理，提升办理效率和服务质量。

### 1.2 模块目标

- **时限管理**：提供灵活的时限配置机制，支持不同站点类型、不同紧急程度的时限设置
- **进度监控**：实时监控信件办理进度，自动判断预警状态
- **主动预警**：自动发送预警提醒，督促相关单位及时处理
- **数据支撑**：为绩效考核、统计分析提供时限数据基础

### 1.3 模块范围

| 功能编号 | 功能名称 | 优先级 | 功能类型 |
|----------|----------|--------|----------|
| F05-01 | 创建预警牌配置 | P1 | 配置管理 |
| F05-02 | 更新预警牌配置 | P1 | 配置管理 |
| F05-03 | 查看预警状态 | P1 | 信息查询 |
| F05-04 | 预警提醒 | P1 | 自动触发 |

### 1.4 与其他模块的关系

| 关联模块 | 关系说明 |
|----------|----------|
| M01-信件核心模块 | 信件创建时自动关联预警牌；信件状态变化触发预警状态计算 |
| M02-信箱管理模块 | 办理单位指派时关联预警时限；办理单位接收预警提醒 |
| M03-审核流程模块 | 流程流转时计算时限进度；审核节点影响时限计算 |
| M09-绩效统计模块 | 统计预警/超期情况，纳入绩效评估 |

## 2. 功能点详细分析

### 2.1 F05-01 创建预警牌配置

#### 功能描述

管理员创建新的预警时限配置，设定不同紧急程度下的办理时限，用于后续信件的时限管理和预警判断。

#### 操作角色

- **主要角色**：管理员
- **权限要求**：需要系统配置权限

#### 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 预警牌名称 | 文本 | 是 | 预警牌显示名称，如"平安厅预警牌" |
| 预警牌编码 | 文本 | 是 | 预警牌唯一编码，如"PINAN" |
| 站点类型 | 选择 | 是 | 适用站点类型，如平安厅/信访/12389/12345等 |
| 工作日时限 | 数字 | 否 | 正常办理时限（工作日） |
| 紧急时限 | 数字 | 否 | 加急办理时限（工作日） |
| 特急时限 | 数字 | 否 | 特急办理时限（工作日） |
| 超紧急时限 | 数字 | 否 | 超紧急办理时限（工作日） |
| 预警开关 | 开关 | 是 | 0关闭/1开启，控制是否启用预警 |

#### 输出

| 输出项 | 说明 |
|--------|------|
| 预警牌编号 | 系统自动生成的唯一标识 |
| 创建成功提示 | 操作成功反馈 |
| 配置详情 | 新创建的预警牌完整配置信息 |

#### 业务规则

| 规则编号 | 规则名称 | 规则说明 |
|----------|----------|----------|
| BR-01 | 编码唯一性 | 预警牌编码必须唯一，不允许重复 |
| BR-02 | 时限合理性 | 时限必须为正整数，紧急时限应小于工作日时限 |
| BR-03 | 站点类型关联 | 预警牌与站点类型一一对应，同一站点类型只能有一个预警牌 |
| BR-04 | 时限递进规则 | 各时限应遵循递进关系：超紧急 < 特急 < 紧急 < 工作日 |
| BR-05 | 默认开关状态 | 新创建的预警牌默认开启预警功能 |

#### 前置条件

1. 用户已登录且具有管理员权限
2. 目标站点类型不存在有效预警牌配置（或需替换）

#### 后置条件

1. 预警牌配置数据写入数据库
2. 后续创建的信件可关联该预警牌

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 编码重复 | 提示"预警牌编码已存在，请使用其他编码" |
| 时限值为负数或零 | 提示"时限必须为正整数" |
| 时限顺序错误 | 提示"紧急时限应小于工作日时限，请检查时限设置" |
| 站点类型已配置 | 提示"该站点类型已存在预警牌，是否需要更新替换？" |
| 权限不足 | 提示"无权限执行此操作"，记录操作日志 |

| 工作日时限 | 数字 | 否 | 新的正常办理时限 |
| 紧急时限 | 数字 | 否 | 新的加急办理时限 |
| 特急时限 | 数字 | 否 | 新的特急办理时限 |
| 超紧急时限 | 数字 | 否 | 新的超紧急办理时限 |

#### 输出

| 输出项 | 说明 |
|--------|------|

| 更新后配置 | 修改后的预警牌配置信息 |

#### 业务规则

| 规则编号 | 规则名称 | 规则说明 |
|----------|----------|----------|

| BR-02 | 时限合理性 | 更新的时限必须为正整数，保持时限递进关系 |
| BR-03 | 即时生效 | 配置变更后立即生效，触发关联信件的预警状态重计算 |
| BR-04 | 操作日志 | 更新操作必须记录日志，包括修改前后的值 |
| BR-05 | 关闭影响 | 关闭预警开关时，关联信件不再触发预警提醒 |

#### 前置条件

2. 目标预警牌配置存在且有效

#### 后置条件

1. 预警牌配置数据更新到数据库
2. 关联信件的预警状态重新计算

#### 异常处理

| 异常场景 | 处理方式 |

| 时限顺序错误 | 提示"时限设置不符合递进规则" |
| 权限不足 | 提示"无权限执行此操作"，记录操作日志 |
| 数据库异常 | 提示"系统异常，请稍后重试"，记录错误日志 |

### 2.3 F05-03 查看预警状态

#### 功能描述

查看指定信件的预警状态，包括当前时限进度、剩余时间、预警级别等详细信息，帮助用户了解信件办理的紧迫程度。

#### 操作角色

- **主要角色**：所有角色（管理员、接收单位、督办单位、主办单位、协办单位）
- **权限范围**：根据用户角色显示相应范围的预警信息

#### 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 信件编号 | 文本 | 是 | 要查看的信件唯一标识 |

#### 输出

| 输出项 | 说明 |
|--------|------|
| 预警状态 | 正常/预警/超期 |
| 预警级别 | 无预警/黄色预警/红色预警 |
| 已用时间 | 从信件创建/指派开始到当前的用时 |
| 剩余时间 | 到时限截止的剩余时间（超期则为负值） |
| 时限截止时间 | 根据配置计算的最晚办理时间 |
| 关联预警牌 | 信件使用的预警牌配置名称 |
| 紧急程度 | 信件当前的紧急级别 |

#### 业务规则

| 规则编号 | 规则名称 | 规则说明 |
|----------|----------|----------|
| BR-01 | 状态计算规则 | 根据信件创建时间、紧急程度、预警牌配置实时计算 |
| BR-02 | 状态判定规则 | 剩余时间>预警阈值：正常；剩余时间≤预警阈值且>0：预警；剩余时间≤0：超期 |
| BR-03 | 工作日计算 | 时限计算只计入工作日，排除法定节假日和周末 |
| BR-04 | 状态展示规则 | 正常显示绿色，预警显示黄色，超期显示红色 |
| BR-05 | 权限过滤 | 根据用户角色过滤可见的信件范围 |

#### 状态判定逻辑

```
预警状态判定算法：
1. 获取信件的紧急程度等级
2. 根据紧急程度选择对应时限（工作日时限/紧急时限/特急时限/超紧急时限）
3. 计算从起始时间到当前的工作日数
4. 计算剩余工作日数 = 时限 - 已用工作日数

状态判定：
- 剩余工作日数 > 时限的20%：正常（绿色）
- 剩余工作日数 ≤ 时限的20% 且 > 0：预警（黄色）
- 剩余工作日数 ≤ 0：超期（红色）
```

#### 前置条件

1. 用户已登录
2. 信件编号有效且用户有查看权限

#### 后置条件

1. 返回预警状态信息
2. 可记录查看日志（可选）

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 信件不存在 | 提示"信件不存在或已删除" |
| 无查看权限 | 提示"无权限查看该信件" |
| 信件未关联预警牌 | 返回"该信件未配置预警时限" |
| 预警牌已关闭 | 返回预警状态为"不监控"或"未配置" |

### 2.4 F05-04 预警提醒

#### 功能描述

系统自动检测超期或即将超期的信件，向相关办理单位发送预警提醒通知，督促及时处理。

#### 操作角色

- **触发方式**：系统自动触发
- **触发时机**：定时任务执行或事件触发

#### 触发条件

| 触发类型 | 触发条件 | 说明 |
|----------|----------|------|
| 定时检测 | 每日固定时间（如上午8点） | 扫描所有在办信件的预警状态 |
| 事件触发 | 信件即将进入预警状态 | 实时检测并推送 |
| 超期触发 | 信件超过办理时限 | 立即发送超期提醒 |

#### 输入

| 输入项 | 来源 | 说明 |
|--------|------|------|
| 信件编号列表 | 系统查询 | 需发送预警的信件集合 |
| 预警类型 | 系统判定 | 预警/超期 |
| 接收单位 | 数据关联 | 信件的办理单位 |

#### 输出

| 输出项 | 说明 |
|--------|------|
| 预警通知 | 发送给相关单位的提醒消息 |
| 通知渠道 | 系统消息/短信/邮件（可配置） |
| 预警记录 | 预警发送日志 |

#### 业务规则

| 规则编号 | 规则名称 | 规则说明 |
|----------|----------|----------|
| BR-01 | 预警阈值 | 达到时限80%（剩余20%）时触发预警提醒 |
| BR-02 | 超期提醒 | 超期后每日提醒一次，直到办理完成 |
| BR-03 | 通知范围 | 通知主办单位、督办单位、接收单位 |
| BR-04 | 免打扰规则 | 已发送预警的信件不重复发送（当日内） |
| BR-05 | 配置开关 | 预警牌开关关闭时不发送预警 |
| BR-06 | 状态排除 | 已完成、已退回等终结状态信件不预警 |

#### 预警通知内容

```
预警通知模板：
【预警提醒】您有以下信件即将超期：
信件编号：[编号]
来信人：[姓名]
标题：[标题]
剩余时间：[X个工作日]
时限截止：[日期]
请及时处理。

超期通知模板：
【超期提醒】您有以下信件已超期：
信件编号：[编号]
来信人：[姓名]
标题：[标题]
超期天数：[X个工作日]
请立即处理。
```

#### 前置条件

1. 系统定时任务正常运行
2. 预警牌配置开启
3. 信件状态为办理中状态

#### 后置条件

1. 预警通知发送成功
2. 预警日志记录写入数据库
3. 办理单位收到预警消息

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 通知发送失败 | 记录失败日志，尝试备用渠道或延迟重发 |
| 接收单位不存在 | 跳过该信件，记录异常日志 |
| 系统任务异常 | 记录错误日志，触发运维告警 |

## 3. 预警规则说明

### 3.1 时限计算规则

#### 起始时间定义

| 起始场景 | 起始时间 | 说明 |
|----------|----------|------|
| 新建信件 | 信件登记时间 | 从系统登记开始计时 |
| 指派办理 | 办理单位指派时间 | 从指派到主办单位开始计时 |
| 退回重办 | 退回操作时间 | 从退回重新办理开始重新计时 |

#### 工作日计算

```
工作日计算规则：
1. 排除周末（周六、周日）
2. 排除法定节假日（从节假日配置表读取）
3. 如有调休工作日，计入工作日
4. 工作日以自然日为单位，不考虑具体工作时间

计算公式：
实际工作日数 = 起始日期到当前日期之间的工作日总数
剩余工作日数 = 配置时限 - 实际工作日数
```

### 3.2 预警级别定义

| 级别 | 状态 | 颜色 | 触发条件 | 提醒频率 |
|------|------|------|----------|----------|
| 正常 | NORMAL | 绿色 | 剩余时间 > 时限×20% | 无提醒 |
| 预警 | WARNING | 黄色 | 剩余时间 ≤ 时限×20% 且 > 0 | 首次预警发送一次 |
| 超期 | OVERDUE | 红色 | 剩余时间 ≤ 0 | 每日提醒一次 |

### 3.3 紧急程度与时限映射

| 紧急程度 | 对应时限字段 | 说明 |
|----------|--------------|------|
| 普通 | 工作日时限 | 正常办理时限 |
| 紧急 | 紧急时限 | 加急办理时限 |
| 特急 | 特急时限 | 特急办理时限 |
| 超紧急 | 超紧急时限 | 超紧急办理时限 |

### 3.4 状态影响规则

| 信件状态 | 时限计算 | 预警判断 |
|----------|----------|----------|
| 待处理(0) | 开始计时 | 正常计算 |
| 受理中(1) | 继续计时 | 正常计算 |
| 待审核(2) | 暂停计时或继续 | 根据配置 |
| 待回复(3) | 停止计时 | 不预警 |
| 已完成(4) | 停止计时 | 不预警 |
| 无法受理(5) | 停止计时 | 不预警 |
| 审核不通过(6) | 重置计时 | 重新计算 |
| 重新修改(9) | 继续或重置 | 根据配置 |
| 重新办理(10) | 重置计时 | 重新计算 |

### 4.2 实体关系图

```
预警牌配置（主配置实体）
    │
    ├── 预警关联（一对多）
    │       │
    │       └── 信件（多对一）
    │               │
    │               ├── 办理单位（预警通知接收方）
    │               │
    │               └── 流转记录（时限计算依据）
    │
    └── 站点类型（多对一，字典关联）

信件实体中相关属性：
    ├── 紧急状态（决定使用哪个时限）
    ├── 站点编号（决定关联哪个预警牌）
    ├── 信件状态（影响是否继续计时）
    └── 登记时间（时限计算起始点）
```

### 4.3 数据流转

```
创建信件时：
    信件.站点编号 → 查找预警牌配置 → 创建预警关联

计算预警状态时：
    预警关联 → 获取预警牌配置 → 获取时限
    信件.紧急状态 → 选择对应时限
    信件.登记时间 + 工作日计算 → 计算已用时间
    时限 - 已用时间 → 剩余时间 → 判断预警状态

发送预警提醒时：
    扫描预警状态为预警/超期的信件
    → 获取办理单位
    → 发送通知
    → 记录预警日志
```

## 5. 用户故事

### US-01：管理员配置预警牌

**作为** 系统管理员
**我想要** 为不同站点类型配置预警时限
**以便于** 系统能自动监控各站点信件的办理时限

**验收条件：**
- 可输入预警牌名称、编码、站点类型
- 可设置工作日时限、紧急时限、特急时限、超紧急时限
- 可控制预警开关的开启/关闭
- 创建后配置立即生效
- 编码不允许重复

### US-02：管理员调整预警配置

**作为** 系统管理员
**我想要** 根据业务需求调整预警时限配置
**以便于** 灵活适应不同时期的办理要求

**验收条件：**
- 可修改时限数值
- 可修改预警开关状态
- 修改后立即生效
- 关联信件的预警状态自动重新计算
- 操作记录日志

### US-03：办理人员查看预警状态

**作为** 主办单位工作人员
**我想要** 在处理信件时查看预警状态
**以便于** 合理安排工作优先级，优先处理紧急信件

**验收条件：**
- 可查看信件的预警状态（正常/预警/超期）
- 可查看剩余时间或超期天数
- 预警状态以颜色区分（绿/黄/红）
- 预警信件在列表中明显标记

### US-04：办理人员接收预警提醒

**作为** 主办单位工作人员
**我想要** 收到即将超期信件的预警提醒
**以便于** 及时处理，避免信件超期

**验收条件：**
- 在信件即将超期时收到系统通知
- 通知包含信件基本信息和剩余时间
- 超期信件每日收到提醒
- 可在消息中心查看预警列表

### US-05：督办人员监控超期信件

**作为** 督办单位工作人员
**我想要** 监督超期信件的办理情况
**以便于** 督促主办单位及时处理，保证办理时效

**验收条件：**
- 可查看所有超期信件列表
- 可对超期信件进行催办
- 可查看各单位的超期统计

## 6. 验收标准

### 6.1 功能验收标准

| 功能编号 | 验收标准 | 验收方法 |
|----------|----------|----------|
| F05-01 | 正常创建预警牌配置，数据写入正确 | 创建测试数据，查询验证 |
| F05-01 | 编码重复时正确提示错误 | 尝试创建重复编码 |
