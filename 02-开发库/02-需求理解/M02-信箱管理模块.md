## 1. 模块概述

### 1.1 模块定位

信箱管理模块是信件处理系统的核心模块之一，负责信件办理单位的指派管理和回复内容管理。该模块连接信件核心模块（M01）和审核流程模块（M03），是信件流转处理的关键枢纽。

### 1.2 模块职责

- **办理单位管理**：为信件指派办理单位，管理主办、督办、协办角色关系
- **回复内容管理**：支持办理单位提交回复内容，管理草稿和正式回复
- **查询导出**：提供办理单位信息的查询和导出功能

### 1.3 业务价值

- 明确责任分工：通过主办/督办/协办角色划分，清晰界定各单位的办理职责
- 提高办理效率：支持快速指派和批量操作，减少人工操作成本
- 确保回复质量：通过草稿保存和审核流程机制，保证回复内容的规范性

### 1.4 模块边界

| 边界项 | 说明 |
|--------|------|
| 上游模块 | M01信件核心模块（信件创建后需要指派办理单位） |
| 下游模块 | M03审核流程模块（回复提交后进入审核流程） |
| 关联数据 | 办理单位、信件回复 |
| 权限范围 | 接收单位可指派办理单位，主办单位可提交回复 |

## 2. 功能点详细分析

### F02-01 创建办理单位

#### 功能描述
为信件指派办理单位，指定其角色（主办/督办/协办），明确信件的办理责任主体。

#### 操作角色
- **主要角色**：接收单位
- **权限要求**：需要有信件指派权限

#### 输入
| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 信件编号 | 编号 | 是 | 待指派的信件唯一标识 |
| 办理单位 | 单位 | 是 | 被指派的办理部门 |
| 单位角色 | 角色 | 是 | 0主办/1督办/2协办 |

#### 输出
| 输出项 | 类型 | 说明 |
|--------|------|------|
| 办理单位记录 | 实体 | 新创建的办理单位记录，包含指派信息 |
| 操作结果 | 结果 | 成功/失败状态及提示信息 |

#### 业务规则
1. **唯一性约束**：同一信件同一部门只能被指派一次，避免重复指派
2. **主办必填**：每封信件必须有且仅有一个主办单位，负责主要办理工作
3. **角色限制**：
   - 主办单位：负责信件的主答复工作，只能有一个
   - 督办单位：负责监督办理进度，可有多个
   - 协办单位：协助办理相关工作，可有多个
4. **状态联动**：指派成功后，信件状态从"待处理"变更为"受理中"

#### 前置条件
1. 信件已创建且处于"待处理"状态
2. 操作用户属于接收单位且有指派权限
3. 待指派的办理单位在系统中有效且可用

#### 异常处理
| 异常场景 | 处理方式 |
|----------|----------|
| 信件不存在 | 返回错误提示"信件不存在"，不允许操作 |
| 信件已被指派主办单位，再次指派主办 | 返回错误提示"该信件已有主办单位"，不允许重复指派 |
| 同一部门重复指派 | 返回错误提示"该部门已被指派"，不允许重复 |
| 办理单位无效或已禁用 | 返回错误提示"办理单位无效"，要求选择有效单位 |
| 用户无权限 | 返回错误提示"权限不足"，拒绝操作 |

### F02-02 更新办理单位

#### 功能描述
修改已指派办理单位的信息，如调整单位角色或更新状态。

#### 操作角色
- **主要角色**：接收单位
- **权限要求**：需要有办理单位管理权限

#### 输入
| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 办理单位记录编号 | 编号 | 是 | 待更新的办理单位记录唯一标识 |
| 更新属性 | 属性集 | 是 | 要更新的属性（角色、状态等） |

#### 输出
| 输出项 | 类型 | 说明 |
|--------|------|------|
| 更新后的办理单位信息 | 实体 | 更新后的完整办理单位记录 |
| 操作结果 | 结果 | 成功/失败状态及提示信息 |

#### 业务规则
1. **角色变更限制**：
   - 主办单位不能变更为其他角色，除非同时指派新的主办单位
   - 协办单位可变更为督办单位，反之亦然
2. **状态更新**：可启用或禁用办理单位，禁用后该单位不再参与办理
3. **操作记录**：所有更新操作需记录修改日志

#### 前置条件
1. 办理单位记录存在
2. 操作用户有管理权限
3. 信件处于可修改状态（未完成）

#### 异常处理
| 异常场景 | 处理方式 |
|----------|----------|
| 办理单位记录不存在 | 返回错误提示"记录不存在" |
| 尝试取消唯一主办单位 | 返回错误提示"必须保留主办单位"，先指派新主办 |
| 信件已完成 | 返回错误提示"信件已办结，不可修改" |
| 无权限 | 返回错误提示"权限不足" |

### F02-03 删除办理单位

#### 功能描述
取消指派办理单位，解除该单位与信件的办理关系。

#### 操作角色
- **主要角色**：接收单位
- **权限要求**：需要有办理单位管理权限

#### 输入
| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 办理单位记录编号 | 编号 | 是 | 待删除的办理单位记录唯一标识 |

#### 输出
| 输出项 | 类型 | 说明 |
|--------|------|------|
| 删除结果 | 结果 | 成功/失败状态及提示信息 |

#### 业务规则
1. **主办保护**：不能删除唯一的主办单位，必须先指派新主办或信件整体作废
2. **状态回退**：取消指派后，若信件无任何办理单位，状态回退至"待处理"
3. **数据保留**：删除为软删除，记录保留用于追溯

#### 前置条件
1. 办理单位记录存在
2. 不是唯一的主办单位（或已指派新主办）
3. 信件未完成

#### 异常处理
| 异常场景 | 处理方式 |
|----------|----------|
| 记录不存在 | 返回错误提示"记录不存在" |
| 删除唯一主办单位 | 返回错误提示"必须保留主办单位" |
| 办理单位已提交回复 | 返回警告提示"该单位已提交回复，确认删除？"，需二次确认 |
| 信件已完成 | 返回错误提示"信件已办结，不可删除" |

### F02-04 批量删除办理单位

#### 功能描述
批量取消指派多个办理单位，提高操作效率。

#### 操作角色
- **主要角色**：接收单位
- **权限要求**：需要有办理单位管理权限

#### 输入
| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 办理单位记录编号列表 | 编号列表 | 是 | 待删除的办理单位记录编号集合 |

#### 输出
| 输出项 | 类型 | 说明 |
|--------|------|------|
| 批量删除结果 | 结果集 | 各记录的删除结果统计 |

#### 业务规则
1. **主办保护**：批量删除时自动跳过主办单位，不执行删除
2. **部分成功**：允许部分记录删除成功，返回详细结果
3. **结果统计**：返回成功数、失败数及失败原因

#### 前置条件
1. 编号列表不为空
2. 操作用户有批量操作权限

#### 异常处理
| 异常场景 | 处理方式 |
|----------|----------|
| 空列表 | 返回错误提示"请选择要删除的记录" |
| 包含主办单位 | 自动跳过主办单位，返回提示"主办单位已跳过" |
| 部分记录不存在 | 跳过不存在记录，返回不存在列表 |

### F02-10 创建信件回复

#### 功能描述
主办单位正式提交信件回复内容，提交后进入审核流程。

#### 操作角色
- **主要角色**：主办单位
- **权限要求**：必须是该信件的主办单位

#### 输入
| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 信件编号 | 编号 | 是 | 信件唯一标识 |
| 办理单位编号 | 编号 | 是 | 提交回复的办理单位记录编号 |
| 回复内容 | 正文 | 是 | 给网民的回复正文 |
| 上级回复内容 | 正文 | 否 | 给上级部门的回复内容 |

#### 输出
| 输出项 | 类型 | 说明 |
|--------|------|------|
| 回复记录 | 实体 | 新创建的回复记录，状态为待审核 |
| 操作结果 | 结果 | 成功/失败状态 |

#### 业务规则
1. **主办限制**：只有主办单位可提交正式回复
2. **内容规范**：
   - 回复内容需符合格式规范
   - 不得包含敏感信息或不当言论
3. **状态变更**：提交后回复状态变为"待审核"，信件状态变为"待审核"
4. **审核流程**：回复内容自动进入审核流程（M03模块）

#### 前置条件
1. 用户是该信件的主办单位
2. 信件处于"受理中"状态
3. 尚未提交正式回复或审核被退回

#### 异常处理
| 异常场景 | 处理方式 |
|----------|----------|
| 非主办单位操作 | 返回错误提示"仅主办单位可提交回复" |
| 信件状态不符 | 返回错误提示"信件状态不允许提交回复" |
| 回复内容为空 | 返回错误提示"回复内容不能为空" |
| 已有待审核回复 | 返回错误提示"已有待审核回复，请等待审核结果" |

### F02-11 创建信件回复草稿

#### 功能描述
保存回复草稿，不提交审核，便于反复修改和完善。

#### 操作角色
- **主要角色**：主办单位
- **权限要求**：必须是该信件的主办单位

#### 输入
| 输出项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 信件编号 | 编号 | 是 | 信件唯一标识 |
| 办理单位编号 | 编号 | 是 | 保存草稿的办理单位记录编号 |
| 回复内容 | 正文 | 否 | 回复草稿内容（允许部分保存） |

#### 输出
| 输出项 | 类型 | 说明 |
|--------|------|------|
| 草稿记录 | 实体 | 新创建或更新的草稿记录 |
| 操作结果 | 结果 | 成功/失败状态 |

#### 业务规则
1. **草稿状态**：保存为草稿状态，不影响信件状态
2. **覆盖机制**：同一信件同一单位只有一份草稿，新保存覆盖旧草稿
3. **不进审核**：草稿不进入审核流程，仅供内部编辑

#### 前置条件
1. 用户是该信件的主办单位
2. 信件未完成

#### 异常处理
| 异常场景 | 处理方式 |
|----------|----------|
| 非主办单位操作 | 返回错误提示"仅主办单位可保存草稿" |
| 信件已完成 | 返回错误提示"信件已办结，不可保存草稿" |
| 草稿保存失败 | 返回错误提示"保存失败"，记录日志 |

| 回复记录编号 | 编号 | 是 | 待修改的回复记录唯一标识 |
| 新回复内容 | 正文 | 是 | 修改后的回复内容 |
| 修改说明 | 说明 | 否 | 修改原因说明 |

#### 输出
| 输出项 | 类型 | 说明 |
|--------|------|------|
| 修改结果 | 结果 | 成功/失败状态 |
| 更新后的回复 | 实体 | 修改后的回复记录 |

#### 业务规则
1. **修改时机**：
   - 草稿状态：可自由修改
   - 待审核状态：审核退回后方可修改
   - 已审核通过：不可修改
2. **已答复保护**：已正式答复网民的回复内容不可修改，确保答复一致性
3. **修改记录**：记录修改历史，保留修改前内容供追溯

#### 前置条件
1. 回复记录存在
2. 回复未正式答复网民
3. 用户是回复提交者

#### 异常处理
| 异常场景 | 处理方式 |
|----------|----------|
| 回复已答复网民 | 返回错误提示"已答复网民，不可修改" |
| 回复审核通过 | 返回错误提示"审核已通过，不可修改" |
| 非回复提交者 | 返回错误提示"仅提交者可修改" |
| 回复记录不存在 | 返回错误提示"回复记录不存在" |

## 4. 用户故事

### 4.1 办理单位管理用户故事

#### US-01：指派主办单位
**作为**接收单位的工作人员
**我想要**为信件指派主办单位
**以便于**明确信件的主要办理责任，推动信件处理流程

**验收条件**：
- 可选择有效的办理单位
- 只能指派一个主办单位
- 指派成功后信件状态变为"受理中"
- 重复指派同一单位时提示错误

#### US-02：指派督办/协办单位
**作为**接收单位的工作人员
**我想要**为信件指派督办或协办单位
**以便于**协调多单位协同办理，监督办理进度

**验收条件**：
- 可指派多个督办或协办单位
- 同一单位不可重复指派为同一角色
- 可查看各单位角色区分

#### US-03：调整办理单位
**作为**接收单位的工作人员
**我想要**修改或取消办理单位指派
**以便于**根据实际情况调整办理分工

**验收条件**：
- 可变更单位角色
- 删除时保护主办单位
- 操作记录可追溯

### 4.2 回复管理用户故事

#### US-04：保存回复草稿
**作为**主办单位的工作人员
**我想要**保存回复草稿反复修改
**以便于**精心准备回复内容，确保回复质量

**验收条件**：
- 草稿可多次保存
- 草稿不影响信件状态
- 草稿不进入审核流程

#### US-05：提交正式回复
**作为**主办单位的工作人员
**我想要**提交正式回复内容
**以便于**完成信件答复工作，进入审核流程

**验收条件**：
- 只有主办单位可提交
- 提交后自动进入审核
- 已有待审核回复时不可重复提交

#### US-06：修改回复内容
**作为**主办单位的工作人员
**我想要**修改被退回的回复内容
**以便于**根据审核意见完善回复

**验收条件**：
- 可修改草稿和退回的回复
- 已答复网民的内容不可修改
- 保留修改历史记录

### 4.3 查询导出用户故事

#### US-07：查看办理单位信息
**作为**相关工作人员
**我想要**查看信件的办理单位信息
**以便于**了解信件的办理分工和进度

**验收条件**：
- 可查看简要信息和详细信息
- 信息按角色分组展示
- 权限控制信息可见范围

#### US-08：导出办理单位数据
**作为**管理人员
**我想要**导出办理单位信息到Excel
**以便于**进行统计分析和汇报

**验收条件**：
- 可按条件筛选导出范围
- 导出格式规范、字段完整
- 大数据量时提供分批建议

## 6. 业务流程说明

### 6.1 办理单位指派流程

```
信件创建 → 接收单位查看 → 选择办理单位 → 指派主办单位 →
  ├── 指派督办单位（可选）
  └── 指派协办单位（可选）
→ 信件状态变为"受理中" → 主办单位开始办理
```

### 6.2 回复提交流程

```
主办单位起草回复 → 保存草稿（可选多次） → 正式提交回复 →
回复状态"待审核" → 进入审核流程（M03） →
  ├── 审核通过 → 准备答复网民
  └── 审核退回 → 修改答复内容 → 重新提交
```
