## 1. 模块概述

### 1.1 模块定位

信件核心模块是gaxx-pro信件处理系统的核心模块，负责信件数据的全生命周期管理，包括信件的创建、查询、修改、删除及恢复等基础操作。该模块为其他模块提供数据基础支撑。

### 1.2 模块目标

- 提供信件数据的完整CRUD操作能力
- 支持信访件的规范化创建与处理流程
- 提供灵活的高级检索能力，满足多维度查询需求
- 实现软删除机制，保障数据可恢复性
- 集成AI模板功能，提升回复效率

### 1.3 核心数据实体

| 实体名称 | 说明 | 关联功能 |
|----------|------|----------|
| 信件 | 信件主信息 | 全部功能 |
| 办理单位 | 信件的办理单位信息 | F01-02, F01-03 |
| 信件回复 | 办理单位提交的回复内容 | F01-03, F01-09 |
| 修改记录 | 信件修改操作记录 | F01-04, F01-05, F01-06 |
| AI模板 | AI自动生成回复的模板配置 | F01-09 |

### 1.4 信件状态流转（接收单位视角）

接收单位视角的核心状态流转：

```
0(待处理) → 1(受理中) → 2(待审核) → 4(已完成)
```

**状态说明：**

| 状态码 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 待处理 | 未指派的留言，信件刚进入系统 |
| 1 | 受理中 | 已指派的留言，正在办理过程中 |
| 2 | 待审核 | 下级单位提交的留言，需要上级单位审核 |
| 4 | 已完成 | 已答复网民的留言 |

## 2. 功能点详细分析

### 2.1 F01-01 创建信件（测试）

#### 功能描述
用于测试环境创建信件数据，方便开发测试人员进行功能验证和数据模拟。

#### 操作角色
- 管理员

#### 输入

| 输入项 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| 来信人姓名 | 姓名 | 是 | 测试用姓名 |
| 来信人身份证 | 身份证号 | 否 | 测试用身份证号 |
| 来信人手机号 | 手机号 | 否 | 测试用手机号 |
| 留言标题 | 标题 | 是 | 信件标题 |
| 留言内容 | 正文 | 是 | 信件详细内容 |
| 数据来源 | 来源 | 是 | 数据来源标识 |
| 信件状态 | 状态 | 否 | 默认为待处理(0) |

#### 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 信件编号 | 编号 | 系统生成的唯一标识 |
| 创建时间 | 时间 | 信件登记时间 |
| 信件信息 | 对象 | 创建成功的信件完整信息 |

#### 前置条件
- 用户已登录且具有管理员角色
- 系统处于测试环境模式
- 数据来源参数已正确配置

#### 业务规则
1. 仅供测试环境使用，正式环境禁止调用
2. 信件编号由系统自动生成，格式遵循编码规则
3. 不触发重复件检测逻辑
4. 不触发预警牌关联逻辑
5. 不进入正式处理流程

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 在正式环境调用 | 返回错误提示，拒绝操作 |
| 必填字段缺失 | 返回参数校验错误信息 |
| 数据格式不正确 | 返回格式校验错误信息 |
| 系统异常 | 记录日志，返回系统错误提示 |

### 2.2 F01-02 创建信访件

#### 功能描述
正式创建信访件，是系统核心入口功能。创建后信件进入待处理状态，等待后续办理流程。

#### 操作角色
- 接收单位

#### 输入

| 输入项 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| 来信人姓名 | 姓名 | 是 | 来信人真实姓名 |
| 来信人身份证 | 身份证号 | 是 | 来信人身份证号码 |
| 来信人手机号 | 手机号 | 是 | 来信人联系电话 |
| 来信人联系地址 | 地址 | 否 | 来信人提供的联系地址 |
| 留言标题 | 标题 | 是 | 信件标题 |
| 留言内容 | 正文 | 是 | 信件详细内容 |
| 留言来源 | 来源 | 是 | 平安厅/信访/12389/12345等 |
| 留言时间 | 时间 | 否 | 来信提交时间 |
| 事发地区 | 地区 | 否 | 事发市（区/县） |
| 事发地区代码 | 代码 | 否 | 事发地区代码 |
| 案发地点 | 地址 | 否 | 案发地点 |
| 留言类型 | 类型 | 否 | 留言类型ID |
| 业务类型 | 类型 | 否 | 业务类型ID |
| 案事件指向 | 类型 | 否 | 案事件指向类型ID |
| 被投诉人姓名 | 姓名 | 否 | 被投诉人姓名 |
| 被投诉人单位 | 单位 | 否 | 被投诉人单位 |
| 紧急状态 | 状态 | 否 | 紧急程度 |
| 附件文件 | 文件 | 否 | 相关附件 |

#### 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 信件编号 | 编号 | 系统生成的唯一标识 |
| 登记时间 | 时间 | 系统登记时间 |
| 信件状态 | 状态 | 固定为待处理(0) |
| 重复件标记 | 标记 | 是否为重复件 |
| 预警牌关联 | 关联 | 关联的预警牌信息 |
| 信件信息 | 对象 | 创建成功的信件完整信息 |

#### 前置条件
- 用户已登录且具有接收单位角色
- 来信人信息完整有效
- 系统字典数据已配置（留言类型、业务类型等）
- 预警牌配置已启用

#### 业务规则

| 规则编号 | 规则描述 |
|----------|----------|
| BR-01 | 必填字段验证：姓名、标题、内容、留言来源为必填项 |
| BR-02 | 身份证号验证：格式校验，并解析归属地信息 |
| BR-03 | 手机号验证：格式校验，符合手机号规则 |
| BR-04 | 重复件检测：根据身份证号+标题相似度算法，自动识别是否为重复件 |
| BR-05 | 预警牌关联：根据站点类型自动关联对应的预警牌配置 |
| BR-06 | 敏感信息加密：身份证号、手机号等敏感信息加密存储 |
| BR-07 | 信件编号生成：按照编码规则自动生成唯一编号 |
| BR-08 | 登记时间记录：系统自动记录登记时间 |

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 必填字段缺失 | 返回参数校验错误信息，列出缺失字段 |
| 身份证号格式错误 | 返回格式错误提示，拒绝创建 |
| 手机号格式错误 | 返回格式错误提示，拒绝创建 |
| 系统字典未配置 | 返回配置错误提示，建议配置字典 |
| 预警牌配置缺失 | 创建信件成功，但不关联预警牌，记录警告日志 |
| 系统异常 | 记录日志，返回系统错误提示，支持重试 |

#### 功能描述
查看信件的完整信息，包括基本信息、办理单位列表、回复内容、办理记录等。

|--------|------|----------|------|

#### 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 信件基本信息 | 对象 | 信件主信息（不含敏感字段） |
| 来信人信息 | 对象 | 来信人信息（按权限脱敏显示） |
| 办理单位列表 | 列表 | 主办、督办、协办单位信息 |
| 回复内容 | 对象 | 当前回复内容信息 |
| 办理记录列表 | 列表 | 办理过程记录 |
| 流程记录列表 | 列表 | 流转处理记录 |
| 标签列表 | 列表 | 关联的标签信息 |
| 预警状态 | 状态 | 当前预警状态（正常/预警/超期） |
| 附件列表 | 列表 | 关联的附件文件信息 |

#### 前置条件
- 用户已登录
- 信件编号有效且存在
- 用户有权限查看该信件

#### 业务规则

| 规则编号 | 规则描述 |
|----------|----------|
| BR-01 | 权限控制：根据用户角色和单位权限显示不同信息范围 |
| BR-02 | 敏感信息脱敏：身份证号、手机号按权限脱敏显示 |
| BR-03 | 非管理员只能查看本单位相关的信件 |
| BR-04 | 已删除信件仅管理员可查看 |
| BR-05 | 自动记录已读状态 |

#### 权限矩阵

| 角色 | 可见信息范围 |
|------|--------------|
| 管理员 | 全部信息 |
| 接收单位 | 本单位接收的信件全部信息 |
| 主办单位 | 本单位主办的信件基本信息+回复内容 |
| 督办单位 | 本单位督办的信件基本信息+办理进度 |
| 协办单位 | 本单位协办的信件基本信息 |

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 信件编号不存在 | 返回404错误，提示信件不存在 |
| 用户无权限 | 返回403错误，提示权限不足 |
| 信件已删除(非管理员) | 返回404错误，提示信件不存在 |
| 系统异常 | 记录日志，返回系统错误提示 |

| BR-04 | 字段校验：更新的属性值需符合数据规范 |
| BR-05 | 已完成信件禁止修改 |

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 信件编号不存在 | 返回404错误，提示信件不存在 |
| 信件已完成 | 返回400错误，提示已完成信件不可修改 |
| 用户无权限 | 返回403错误，提示权限不足 |
| 更新属性非法 | 返回400错误，列出非法属性 |
| 属性值格式错误 | 返回400错误，提示格式错误 |
| 系统异常 | 记录日志，返回系统错误提示 |

### 2.5 F01-05 删除信件

#### 功能描述
删除单个信件，采用软删除机制，数据可恢复。

#### 操作角色
- 管理员

#### 输入

| 输入项 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| 信件编号 | 编号 | 是 | 信件唯一标识 |
| 删除原因 | 原因 | 是 | 删除操作原因说明 |

#### 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 删除结果 | 状态 | 删除成功/失败 |
| 删除时间 | 时间 | 删除操作时间 |

#### 前置条件
- 用户已登录且具有管理员角色
- 信件编号有效且存在
- 信件未被删除

#### 业务规则

| 规则编号 | 规则描述 |
|----------|----------|
| BR-01 | 软删除机制：仅标记删除状态，数据物理保留 |
| BR-02 | 删除原因必填：删除操作必须记录原因说明 |
| BR-03 | 记录删除日志：记录删除时间、删除人、删除原因 |
| BR-04 | 关联数据处理：关联的办理单位、回复等数据保留但标记关联信件已删除 |
| BR-05 | 状态记录：记录删除前的信件状态，用于恢复时还原 |

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 信件编号不存在 | 返回404错误，提示信件不存在 |
| 信件已删除 | 返回400错误，提示信件已删除 |
| 删除原因未填 | 返回400错误，提示需填写删除原因 |
| 用户无权限 | 返回403错误，提示权限不足 |
| 系统异常 | 记录日志，返回系统错误提示 |
