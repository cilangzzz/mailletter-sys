## 1. 模块概述

### 1.1 模块定位

绩效统计模块是gaxx-pro信件处理系统的数据分析与决策支持模块，为管理层提供信件办理效率、回复质量和网民满意度三个维度的统计分析功能。该模块通过对信件处理全过程数据的聚合分析，帮助管理层评估各单位工作绩效、发现业务瓶颈、优化资源配置。

### 1.2 业务价值

- **效率监控**：实时跟踪各单位信件处理效率，及时发现超期风险
- **质量评估**：量化回复审核通过率，推动回复质量提升
- **满意度分析**：汇总网民评价反馈，衡量服务满意度水平
- **决策支持**：为绩效考核、资源调配、流程优化提供数据依据

### 1.3 模块优先级

**P2** - 属于运营支撑类功能，在核心业务稳定运行后实施。

### 1.4 功能清单

| 功能编号 | 功能名称 | 优先级 | 说明 |
|----------|----------|--------|------|
| F09-01 | 办理效率统计 | P2 | 统计各单位信件办理效率指标 |
| F09-02 | 回复质量统计 | P2 | 统计回复审核通过率等质量指标 |
| F09-03 | 满意度统计 | P2 | 统计网民满意度评价数据 |

## 2. 功能点详细分析

### 2.1 F09-01 办理效率统计

#### 功能描述

统计各单位信件办理效率，包括办理时效、超期情况、延期申请、催办次数等指标，生成办理效率统计报表，支持管理层评估各单位处理效率和发现效率瓶颈。

#### 操作角色

| 角色 | 权限范围 |
|------|----------|
| 管理层 | 可查看本单位及下级单位的效率统计 |
| 接收单位管理员 | 可查看本单位效率统计 |
| 系统管理员 | 可查看全部单位效率统计 |

#### 输入参数

| 参数名称 | 参数类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 统计周期 | 时间范围 | 是 | 统计时间范围（如本月、本季度、自定义） |
| 单位范围 | 单位列表 | 是 | 选择统计的单位范围（单个/多个/全部） |
| 信件状态筛选 | 状态列表 | 否 | 按信件状态筛选（已完成/进行中/全部） |
| 紧急状态筛选 | 状态列表 | 否 | 按紧急程度筛选（普通/加急/特急/超紧急） |
| 数据来源筛选 | 来源列表 | 否 | 按信件来源筛选（平安厅/信访/12389等） |

#### 输出内容

**办理效率统计报表**，包含以下统计指标：

| 指标名称 | 计算方式 | 说明 |
|----------|----------|------|
| 受理信件数 | COUNT(信件) | 统计周期内受理的信件总数 |
| 已办结数 | COUNT(信件 WHERE 状态=已完成) | 已办结信件数量 |
| 办结率 | 已办结数 / 受理信件数 × 100% | 信件办结比例 |
| 平均办理时长 | AVG(办结时间 - 登记时间) | 平均每封信件办理耗时 |
| 最短办理时长 | MIN(办结时间 - 登记时间) | 最快办结耗时 |
| 最长办理时长 | MAX(办结时间 - 登记时间) | 最慢办结耗时 |
| 超期信件数 | COUNT(流转记录 WHERE 是否超期=是) | 超期未办结信件数量 |
| 超期率 | 超期信件数 / 受理信件数 × 100% | 超期信件占比 |
| 平均超期天数 | AVG(超期天数) WHERE 是否超期=是 | 超期信件平均超期天数 |
| 延期申请次数 | SUM(延期次数) | 申请延期总次数 |
| 被催办次数 | SUM(催办次数) | 被催办总次数 |

#### 业务规则

1. **时间计算规则**
   - 办理时长按工作日计算，排除法定节假日
   - 紧急信件按紧急时限标准计算是否超期
   - 普通信件按工作日时限标准计算是否超期

2. **统计范围规则**
   - 只统计已进入办理流程的信件（状态不为待处理）
   - 退回重办的信件重新计算办理时间
   - 无法受理的信件不计入办结率统计

3. **单位统计规则**
   - 主办单位负责该信件的效率统计
   - 协办单位单独统计协办部分的效率
   - 督办单位统计督办处理时效

4. **数据聚合规则**
   - 支持按单位层级汇总（单位→处室→市局→厅级）
   - 支持按时间维度汇总（日→周→月→季度→年）
   - 支持按信件类型汇总（留言类型/业务类型）

#### 前置条件

1. 用户具有管理层角色权限
2. 系统中存在可统计的信件数据
3. 预警牌配置已设置时限标准

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 统计周期无数据 | 显示"暂无数据"，不显示报表 |
| 单位范围权限不足 | 提示"无权限查看该单位数据"，自动过滤 |
| 统计计算超时 | 提示"数据量较大，正在计算"，支持后台异步计算 |
| 数据导出失败 | 提示"导出失败，请稍后重试"，记录异常日志 |

### 2.2 F09-02 回复质量统计

#### 功能描述

统计各单位回复内容的审核通过率、退回修改率等质量指标，结合审核环节的评分和评价意见，生成回复质量统计报表，帮助管理层评估回复撰写质量。

#### 操作角色

| 角色 | 权限范围 |
|------|----------|
| 管理层 | 可查看本单位及下级单位的回复质量统计 |
| 督办单位 | 可查看主办单位的回复质量统计（督办范围内） |
| 接收单位管理员 | 可查看本单位回复质量统计 |
| 系统管理员 | 可查看全部单位回复质量统计 |

#### 输入参数

| 参数名称 | 参数类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 统计周期 | 时间范围 | 是 | 统计时间范围 |
| 单位范围 | 单位列表 | 是 | 选择统计的单位范围 |
| 审核阶段筛选 | 阶段列表 | 否 | 一审/终审/全部 |
| 信件类型筛选 | 类型列表 | 否 | 按留言类型/业务类型筛选 |
| 退回类型筛选 | 类型列表 | 否 | 重新修改/重新办理/审核不通过 |

#### 输出内容

**回复质量统计报表**，包含以下统计指标：

| 指标名称 | 计算方式 | 说明 |
|----------|----------|------|
| 提交回复数 | COUNT(信件回复 WHERE 状态=已提交) | 提交审核的回复总数 |
| 一审通过数 | COUNT(流程 WHERE 流程类型=一审通过) | 一审审核通过次数 |
| 一审通过率 | 一审通过数 / 提交回复数 × 100% | 一审通过比例 |
| 终审通过数 | COUNT(流程 WHERE 流程类型=终审通过) | 终审审核通过次数 |
| 终审通过率 | 终审通过数 / 一审通过数 × 100% | 终审通过比例 |
| 整体通过率 | 终审通过数 / 提交回复数 × 100% | 从提交到通过的完整比例 |
| 退回修改次数 | COUNT(流程 WHERE 流程类型=重新修改) | 被退回要求修改的次数 |
| 退回重办次数 | COUNT(流程 WHERE 流程类型=重新办理) | 被退回要求重办的次数 |
| 审核不通过次数 | COUNT(流程 WHERE 流程类型=审核不通过) | 审核驳回次数 |
| 平均修改轮次 | AVG(修改次数) | 平均每封信件修改提交次数 |
| 平均审核时长 | AVG(审核时间 - 提交时间) | 审核环节平均耗时 |

#### 业务规则

1. **审核流程统计规则**
   - 一审由督办单位执行，终审由接收单位执行
   - 直接终审通过的按终审通过统计
   - 多轮修改的回复累计统计修改次数

2. **退回类型分类**
   - 重新修改：回复内容需调整，无需重新调查
   - 重新办理：需重新开展调查办理
   - 审核不通过：回复完全不通过，需重新撰写

3. **质量评分规则**
   - 流转记录中的质量评分纳入统计（如有）
   - 质量评价意见分类汇总高频问题
   - AI生成模板使用率可作为参考指标

4. **时间计算规则**
   - 审核时长从提交时间到审核结果时间
   - 多轮审核累计计算总审核时长

#### 前置条件

1. 用户具有管理层角色权限
2. 审核流程已执行并有审核结果
3. 信件回复表中有提交记录

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 统计周期内无提交记录 | 显示"暂无数据"，不显示报表 |
| 审核流程数据不完整 | 标注"数据不完整"，按已有数据计算 |
| 权限不足 | 自动过滤无权限单位数据 |
| 比率计算分母为零 | 显示"-",不计算比率 |

### 2.3 F09-03 满意度统计

#### 功能描述

统计网民对信件办理的满意度评价数据，包括整体满意度及各分项满意度（诉求解决、响应速度、服务态度、办事能力、跟进服务），生成满意度统计报表，帮助管理层了解服务质量和群众满意度水平。

#### 操作角色

| 角色 | 权限范围 |
|------|----------|
| 管理层 | 可查看本单位及下级单位的满意度统计 |
| 接收单位管理员 | 可查看本单位满意度统计 |
| 系统管理员 | 可查看全部单位满意度统计 |

#### 输入参数

| 参数名称 | 参数类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 统计周期 | 时间范围 | 是 | 统计时间范围 |
| 单位范围 | 单位列表 | 是 | 选择统计的单位范围 |
| 满意度维度筛选 | 维度列表 | 否 | 整体/诉求解决/响应速度/服务态度/办事能力/跟进服务 |
| 信件类型筛选 | 类型列表 | 否 | 按留言类型/业务类型筛选 |
| 是否回访筛选 | 标记 | 否 | 已回访/未回访 |

#### 输出内容

**满意度统计报表**，包含以下统计指标：

| 指标名称 | 计算方式 | 说明 |
|----------|----------|------|
| 已评价信件数 | COUNT(信件 WHERE 满意度!=空) | 已有满意度评价的信件数 |
| 回访信件数 | COUNT(信件办理 WHERE 是否回访=是) | 已进行回访的信件数 |
| 回访率 | 回访信件数 / 已办结数 × 100% | 回访覆盖比例 |
| 整体满意数 | COUNT(整体满意度=满意) | 整体满意的信件数 |
| 整体基本满意数 | COUNT(整体满意度=基本满意) | 整体基本满意的信件数 |
| 整体不满意数 | COUNT(整体满意度=不满意) | 整体不满意的信件数 |
| 整体满意度 | (满意数 + 基本满意数×0.5) / 已评价数 × 100% | 综合满意度评分 |
| 诉求解决满意度均值 | AVG(诉求解决满意度) | 诉求解决维度平均评分 |
| 响应速度满意度均值 | AVG(响应速度满意度) | 响应速度维度平均评分 |
| 服务态度满意度均值 | AVG(服务态度满意度) | 服务态度维度平均评分 |
| 办事能力满意度均值 | AVG(办事能力满意度) | 办事能力维度平均评分 |
| 跟进服务满意度均值 | AVG(跟进服务满意度) | 跟进服务维度平均评分 |

#### 业务规则

1. **满意度评价来源**
   - 满意度数据来自信件办理表的回访记录
   - 满意度选项：满意、基本满意、不满意
   - 部分信件可能无满意度评价（未回访或无法回访）

2. **满意度计算规则**
   - 综合满意度采用加权算法（满意100分、基本满意50分、不满意0分）
   - 各分项满意度若有数值评分，计算平均值
   - 无评价的信件不计入满意度统计

3. **回访统计规则**
   - 回访方式包括电话回访、当面回访等
   - 未回访需记录原因
   - 群众电话回访反映内容可分类统计

4. **评价趋势分析**
   - 支持按时间趋势展示满意度变化
   - 支持与历史同期对比分析
   - 不满意信件可追溯查看详情

#### 前置条件

1. 用户具有管理层角色权限
2. 信件办理表中有回访和满意度记录
3. 部分信件已办结并完成回访

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 统计周期内无满意度数据 | 显示"暂无数据"，提示回访覆盖率不足 |
| 某维度无评分数据 | 该维度显示"-",不计算均值 |
| 满意度评价数据不完整 | 按已有数据统计，标注数据完整度 |
| 权限不足 | 自动过滤无权限单位数据 |

## 3. 统计指标定义

### 3.1 办理效率指标体系

| 指标类别 | 指标名称 | 计算公式 | 数据来源 | 统计频率 |
|----------|----------|----------|----------|----------|
| **时效指标** | 平均办理时长 | AVG(办结时间 - 登记时间) | 信件表 | 日/周/月 |
| | 最短办理时长 | MIN(办结时间 - 登记时间) | 信件表 | 月 |
| | 最长办理时长 | MAX(办结时间 - 登记时间) | 信件表 | 月 |
| **完成指标** | 办结率 | 已办结数 / 受理信件数 × 100% | 信件表 | 日/周/月 |
| | 受理信件数 | COUNT(信件 WHERE 状态!=待处理) | 信件表 | 日 |
| | 已办结数 | COUNT(信件 WHERE 状态=已完成) | 信件表 | 日 |
| **超期指标** | 超期率 | 超期信件数 / 受理信件数 × 100% | 流转记录表 | 日/周 |
| | 平均超期天数 | AVG(超期天数) WHERE 是否超期=是 | 流转记录表 | 周/月 |
| | 超期信件数 | COUNT(流转记录 WHERE 是否超期=是) | 流转记录表 | 日 |
| **延期指标** | 延期申请次数 | SUM(延期次数) | 流转记录表 | 月 |
| | 延期率 | 延期信件数 / 受理信件数 × 100% | 流转记录表 | 月 |
| **催办指标** | 被催办次数 | SUM(催办次数) | 流转记录表 | 月 |
| | 催办率 | 被催办信件数 / 受理信件数 × 100% | 流转记录表 | 月 |

### 3.2 回复质量指标体系

| 指标类别 | 指标名称 | 计算公式 | 数据来源 | 统计频率 |
|----------|----------|----------|----------|----------|

| | 退回重办率 | 退回重办次数 / 提交回复数 × 100% | 信件流程表 | 月 |
| | 审核驳回率 | 审核不通过次数 / 提交回复数 × 100% | 信件流程表 | 月 |
| **修改指标** | 平均修改轮次 | AVG(修改次数) | 信件回复表 | 月 |

### 3.3 满意度指标体系

| | 响应速度满意度 | AVG(响应速度满意度) | 信件办理表 | 月 |
| | 服务态度满意度 | AVG(服务态度满意度) | 信件办理表 | 月 |
| | 办事能力满意度 | AVG(办事能力满意度) | 信件办理表 | 月 |

## 4. 数据实体关联

### 4.1 核心数据实体

绩效统计模块涉及以下数据实体：

| 实体名称 | 用途 | 关键属性 |
|----------|------|----------|
| 信件 | 效率统计基础数据 | 信件状态、登记时间、办结时间、办理单位 |
| 办理单位 | 单位维度统计数据 | 办理单位、单位角色、答复时间、办理状态 |
| 信件回复 | 质量统计基础数据 | 回复单位、状态、是否答复网民 |
| 信件办理 | 满意度统计基础数据 | 是否回访、各维度满意度 |
| 流转记录 | 效率补充数据 | 是否超期、超期天数、延期次数、催办次数 |
| 信件流程 | 质量补充数据 | 流程类型、流程状态码、处理时间 |

### 4.2 数据关联关系

```
绩效统计数据关联图：

信件（主统计实体）
    │
    ├── 办理单位 ────── 单位维度统计
    │       │
    │       ├── 信件回复 ────── 质量统计（通过率、修改次数）
    │       │
    │       ├── 信件办理 ────── 满意度统计（回访、满意度）
    │       │
    │       ├── 流转记录 ────── 效率补充（超期、延期、催办）
    │       │
    │       └── 信件流程 ────── 质量补充（审核通过/退回）
    │
    └── 预警关联 ────── 时限标准参照
            │
            └── 预警牌配置 ────── 超期判定基准
```

### 4.3 统计数据映射

| 统计功能 | 主要数据实体 | 关键统计字段 |
|----------|--------------|--------------|
| 办理效率统计 | 信件、流转记录 | 信件状态、登记时间、办结时间、是否超期、超期天数、延期次数 |
| 回复质量统计 | 信件回复、信件流程 | 回复状态、流程类型（审核通过/退回）、处理时间 |
| 满意度统计 | 信件办理 | 是否回访、整体满意度、诉求解决满意度、响应速度满意度等 |

## 7. 非功能性需求

### 7.1 性能要求

| 指标 | 要求 |
|------|------|
| 统计计算响应时间 | 小于30秒（数据量小于10万条） |
| 大数据量处理 | 支持异步计算，计算完成后通知用户 |
| 报表导出时间 | 小于60秒（数据量小于5万条） |
| 并发用户数 | 支持10个用户同时查询统计 |

### 7.2 安全要求

- 统计数据按角色权限过滤，无权限数据不可见
- 导出的报表文件按单位隔离存储
- 统计操作记录审计日志

### 7.3 可用性要求

- 统计界面操作简单，三步完成查询（选择周期→选择单位→查看报表）
- 报表支持图表可视化展示
- 异常情况有明确的提示信息
