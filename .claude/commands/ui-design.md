# 设计页面UI

根据产品UI设定规范，为指定页面补全完整的UI设计信息。

## 使用方法
```
/ui-design <页面标识或页面名称>
```

## 执行步骤

### 1. 读取产品UI设定规范
首先读取 `02-开发库/03-系统设计/前端/产品UI设定.md`，获取完整的UI设计规范。

### 2. 收集页面信息
根据用户指定的页面标识或名称，在项目中查找：
- 相关的需求理解文档（`02-开发库/02-需求理解/`目录）
- 相关的需求开发文档（`02-开发库/01-需求开发/`目录）
- 数据属性定义（`02-开发库/01-需求开发/数据属性.md`）

从这些文档中提取：
- 页面功能描述
- 涉及的数据字段
- 用户操作场景
- 业务规则约束

### 3. 生成页面UI设计文档
按照产品UI设定规范的章节结构，生成完整的页面UI设计文档，包含：

#### 基本信息
- 页面名称：用户可见标题
- 页面标识：路由标识符
- 所属模块：模块归属
- 页面类型：列表页/详情页/表单页等

#### 页面布局
- 描述选用的布局框架
- 各区域的内容安排

#### 页面元素清单
- 导航元素（面包屑、标签页）
- 功能按钮（类型、位置、触发条件）
- 数据展示组件（表格列配置、卡片字段）
- 表单控件（控件类型、验证规则）
- 状态提示（加载、空数据、错误）

#### 交互规则
- 核心用户操作流程
- 按钮点击响应
- 表单验证与提交
- 弹窗触发条件

#### 数据字段表
按规范格式列出所有字段：
| 字段名称 | 字段标识 | 数据类型 | 是否必填 | 是否可编辑 | 显示格式 |

#### 样式说明
- 特殊样式需求
- 响应式适配要点

### 4. 输出位置
生成的页面UI设计文档应保存在：
`02-开发库/03-系统设计/前端/<模块标识>-<页面标识>.md`

例如：`02-开发库/03-系统设计/前端/M01-letter-list.md`

## 示例

用户输入：
```
/ui-design 信件列表页
```

输出：
1. 读取产品UI设定规范
2. 查找M01模块相关需求文档
3. 生成 `M01-letter-list.md` 页面UI设计文档
